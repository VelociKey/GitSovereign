# ADR-014: Firehorse Sovereign Transport (QUIC + jeBNF)

## Status
Proposed (Pulse 2 - GitSovereign)

## Context
GitSovereign requires a high-performance, low-latency, and auditable transport layer for the "Zero-Disk" harvest and rehydration of codebase assets. ADR-013 proposed ConnectRPC + Cronet, which introduced significant overhead in serialization and HTTP/2 framing for raw binary streams (git bundles).

## Decision
We will supersede ADR-013 with the **Firehorse Sovereign Transport**, utilizing raw **QUIC (UDP)** and **jeBNF** grammatical framing.

1.  **Transport Protocol**: Raw QUIC (via `quic-go-0.50.0`) provides sub-second session establishment and stream multiplexing without the overhead of HTTP/3 layers.
2.  **Framing**: `jeBNF SegmentHeader` framing allows for real-time auditability and forensic traceability of binary segments without full stream re-parsing.
3.  **Security**: Mutual TLS (mTLS) with SPIFFE-derived SVIDs for identity-anchored transport.
4.  **Zero-Disk Pipeline**: Git bundles are piped directly from memory to the QUIC stream, bypassing local disk I/O.

## Consequences
- **Pros**: 
    - Sub-second RTO (Recovery Time Objective).
    - Reduced CPU overhead for large binary exfiltration.
    - Native support for parallelism at the stream level.
- **Cons**:
    - Requires UDP egress permission in corporate environments.
    - Custom framing requires specialized audit tools (Sentry-Agent).

## Verification
- Verified by **Pinnacle-Assurance** reports.
- Hardened by the **ControlPlane** worker pool density logic.
